WorldWide Movie Reference

Documentation

This web application will allow users to search for information on their favourite films, actors, or movie genres.

Additional functionality will allow personal information to be saved, as well as the ability to compile film lists etc.

Request URL HTTP Method Request Body Data Format Response Body Data Format
Return a list of All movies /movies GET None A JSON object holding data about all movies
Return data about a single movie using the name /movies/[Name] GET None A JSON object holding data about a specific movie, containing the title, movie length, year of release, a description of the film, its genre, the director and a relevant image.

Example:

{
"_id" : ObjectId("5df39780ed3766a45674006f"),
"title" : "Raiders of the Lost Ark",
"year" : 1981,
"description" : "In 1936, archaeologist and adventurer Indiana Jones is hired by the U.S. government to find the Ark of the Covenant before Adolf Hitler's Nazis can obtain its awesome powers.",
"genre" : "Adventure",
"director" : "Steven Spielberg",
"actors" : [],
"length" : 115,
"imageUrl" : "raidersofthelostark.png"
}

Return data about a genre using the name /genres/[Name] GET None A JSON object holding data about a genre, containing its name and description.

Example:

{
"name": "Action",
"description": "Action film is a film genre in which the protagonist or protagonists are thrust into a series of events that typically include violence, extended fighting, physical feats, and frantic chases." }

Return data about a director using their name /directors/[Name] GET None A JSON object holding data about a director, containing their name, a bio, their year of birth and year of death.

Example:

{
"name" : "Alfred Hitchcock",
"bio" : "Sir Alfred Joseph Hitchcock, KBE was an English film director and producer, widely regarded as one of \ the most influential and extensively studied filmmakers in the history of cinema. His films are marked by a macabre \ sense of humour and a somewhat bleak view of the human condition.",
"birthYear" : "1899",
"deathYear" : "1980"
}

Allow new users to register /users POST A JSON object containing the relevant data needed to register.

Example:

{
"name" : "Harry Larry", "username": "harrylarry1962",
"password": "easytoguesspassword123",
"email": "harrylarry62@aol.com",
"birthday": "1962-08-08",
}

A JSON object holding data about the new user, including an ID and an empty array of potential favourite movies.

Example:

{
"_id" : ObjectId("5dfa16eda4fd6e0ec5e4fb51"),
"favouriteMovies" : [],
"name" : "Harry Larry",
"username": "harrylarry1962",
"password": "easytoguesspassword123",
"email": "harrylarry62@aol.com",
"birthday": "1962-08-08",
}

Allow users to update their personal information /users/[Username] PUT A JSON object holding the desired data updates.

Example:

{
"_id" : ObjectId("5dfa16eda4fd6e0ec5e4fb51"),
"favouriteMovies" : [],
"name" : "Harry Larry",
"username": "harrylarry1965",
"password": "easytoguesspassword123",
"email": "harrylarry65@aol.com",
"birthday": "1965-03-17",
}

A JSON object displaying the data updates made.

Example:

{
"_id" : ObjectId("5dfa16eda4fd6e0ec5e4fb51"),
"favouriteMovies" : [],
"name" : "Harry Larry",
"username": "harrylarry1965",
"password": "easytoguesspassword123",
"email": "harrylarry65@aol.com",
"birthday": "1965-03-17",
}

Allow users to add a movie to their list of favourites /users/[Username]/movies/[MovieID] POST None A JSON object hllding the user data, including the new favourite movie ID.

Example:

{
"_id" : ObjectId("5dfa16eda4fd6e0ec5e4fb51"),
"favouriteMovies" : [
"5df39696ed3766a456740066"
],
"name" : "Harry Larry",
"username": "harrylarry1965",
"password": "easytoguesspassword123",
"email": "harrylarry65@aol.com",
"birthday": "1965-03-17",
}

Allow users to remove a movie from their favourite list. /users/[Username]/movies/[MovieID] DELETE None A JSON object holding the users data, with the specified movie ID removed.

Example:

{
"_id" : ObjectId("5dfa16eda4fd6e0ec5e4fb51"),
"favouriteMovies" : [],
"name" : "Harry Larry",
"username": "harrylarry1965",
"password": "easytoguesspassword123",
"email": "harrylarry65@aol.com",
"birthday": "1965-03-17",
}

Allow a user to remove all their personal data (de-register) /users/[Username] DELETE None A message indicating that personal information has been successfully removed.
